Glossary

Branch - A different version of the codebase permits feature developments; these are entirely independent. Clone - Each local repository holds one copy. Commit - A detailed snapshot of complete changes made in a repository. Fetch - The updates are retrieved from a remote repository, but they are not merged. GIT - A version control system tracks every single change in the code. Github - It is a platform. It handles cloud Git repositories. Merge - Brings together changes from separate branches. Merge Conflict - A conflict occurs if different branches contain changes that are incompatible. Push - Every local commit is sent to all remote repositories. Pull - Retrieves changes coming from another repository and carefully puts them together. Remote - The repository is stored on a server like GitHub. Repository - A storage location for past versions, along with project files.
